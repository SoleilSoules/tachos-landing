- **Паддинги** контейнеров секций: паттерн `px-5 sm:px-8 lg:px-[80px]` (Footer `px-5 … lg:px-[96px]`).
- **Cases** `grid-cols-1 sm:grid-cols-2`; **Products** планшет-мокап `hidden lg:block`, switcher в столбик, колода ниже; крупные заголовки секций → `clamp`; **Blog/Services/Reviews** паддинги+заголовки; **LetterBody** письмо `text-[17px] lg:text-[20px]`.

- **МЕТОД проверки мобилы:** обычный headless `--window-size=390` рендерит на **vw=500** (артефакт new-headless) → скрин обрезается и врёт. Правильно — **CDP `Emulation.setDeviceMetricsOverride {width:390}`** (нужен `ws`, `npm i ws --no-save`); тогда vw=390 и `scrollWidth==390` = нет оверфлоу. Скрипт-образец гонялся в этой сессии.

## МЕТОД РАБОТЫ (так идёт этот раунд — придерживаться)
Гоша **смотрит localhost вживую** и присылает правки по одной (часто скрин секции). Цикл: правка → точечный фикс → **самопроверка скриншотом (headless Chrome)** → Гоша смотрит в Safari → «ок / докрутить». Маленькими шагами, развилки/вкусовое — на его выбор.
- Самопроверка скрином: headless Chrome 148+ **виснет после --screenshot** → запускать в фоне (`run_in_background`), ждать файл по размеру (>120КБ), потом `kill`; в macOS нет `timeout`. Кроп через `ffmpeg -vf crop=W:H:x:y`. Флаги: `--headless=new --hide-scrollbars --force-prefers-reduced-motion --run-all-compositor-stages-before-draw --user-data-dir=/tmp/...`. Полная страница: `--window-size=1440,1340`.
- Компиляцию проверять: `curl -s localhost:3000` + grep dev-лога (`/tmp/tachos-dev.log`) на `error|⨯`. Типы — `npx tsc --noEmit`.
